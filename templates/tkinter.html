Python by default is a text base programming language, for this reason if we wanna have graphical user interface with it, we need to use extra framework "module".

There are many choices for Gui programming in python, for example: WxPython, PyGui, Pyside, Tkinter, Pyqt and ...

Tkinter have some pros and cons:
pros:1) Its part of the standard library so you dont need to download any extra module.
2) Have large community, its simple and many programmers use it even for writing some basic games.
3) Its cross platform you can use it on windows, linux and mac and it worth to mention that written by Fredrik Lundh so thanks Fredrik.

Cons: 1) Its ugly, well thats what some programmers say a bout it but its a mater of personal taste.
2) It doesn't have pythonic look!
3) its not naturally part of the python, its just a wrapper around the TK and Tci module which is interpreted embedded in python interpreter.
But worth to mention that Tkinter can be used by other languages as well so thats some a plus. in Tkinter we are dealing with Widgets and events same as Pygame or Turtle.
Because every program that have constant interaction with user is Event Driven Program. as i mentioned before Tkinter is part of python standard library, so the only thing we need to do is to import it. In first example we are going to create a main window. same as what we do when working with Pygame or turtle to create the screen. so we create a main window and call the main loop to keep it up and running.


Its very possible to use only functions to create a gui by tkinter but it can be a good idea to use Classes as our programs grows so we can go by DRY principle which means Dont repeat yoursel.
